## 1. 目標與範圍

### 1.1 目標

提供「Dataset Item」的完整生命週期管理，作為 LoRA fine-tuning 訓練資料的最終來源：

1. 從外部標註檔匯入（4 種格式：JSON / XLSX / SRT / TXT）。
2. 從已完成的 Job 一鍵建立（自動複製 audio + segments → 訓練 JSON 格式）。
3. 重用 M3 的 TranscriptEditor 編輯 dataset segments（時間軸 / 波形 / 文字 / 講者）。
4. 匯出 3 種格式（JSON / SRT / XLSX）給人工複核或匯入其他工具。
5. 提供 4 種範本下載（json / xlsx / srt / txt）給標註人員使用。

### 1.2 不在本里程碑範圍

| 項目 | 移交里程碑 / 備註 |
|---|---|
| CSV / VTT 匯入 | Backlog（與 XLSX / SRT 高度重疊） |
| Dataset bulk select → 加入訓練的跳轉行為 | M4（Training 頁尚未實作，無落點） |
| Dataset 前端自動測試 | Backlog B1（與 M3 同一節奏） |
| audio 去重 / hashing 比對 | Backlog（內測階段不需要） |
| audio 版本歷史 | Backlog（label 版本歷史也不做，沿用 updated_at） |

### 1.3 設計決策摘要

| # | 主題 | 結論 |
|---|---|---|
| 1 | DatasetItem 編輯 UI | 重用 TranscriptEditor，抽 `EditorSource` 介面 |

### 2.2 Speaker 索引慣例（CLAUDE.md §4.3 補充）

| 上下文 | 格式 |
|---|---|

```python
async def create_from_import(
    db, *,
    project_id: int,
    audio_upload: UploadFile,
    label_upload: UploadFile,
    format: str,
) -> DatasetItem:
    """
    流程：
    1. validate format ∈ {json, xlsx, srt, txt}
    2. 把 audio 與 label 各自 stream 寫到 tempfile（tempfile.NamedTemporaryFile，
       importer 與 mutagen 都吃 file path）

    """

async def create_from_job(
    db, *,
    job_id: str,
    notes: str | None,
) -> DatasetItem:
    """
    流程：
    1. 取 Job + segments；status 必須為 SUCCEEDED → 否則 AppError(INVALID_JOB_STATE)
    2. ext = Path(job.audio_path).suffix.lstrip(".").lower()
    3. label = build_label_from_job(job, segments)：
       - audio_duration = job.duration_sec
       - audio_path = f"audio.{ext}"
       - segments = [{speaker: s.speaker_id - 1, text: s.text, start: s.start_time, end: s.end_time} ...]
       - customized_context = project.hotwords
    4. INSERT row（source=FROM_JOB, source_job_id=job.id, audio_path=""），commit 取 item_id
    5. dst_key = f"datasets/{item_id}/audio.{ext}"
       file_store.copy(job.audio_path, dst_key)   # job.audio_path 已是 file_store 相對 key
    6. UPDATE row.audio_path = dst_key
    7. 失敗回滾：刪 row + rmtree datasets/{item_id}/
    """

async def update_item(
    db, *, item_id: int, patch: DatasetItemPatch,

    1. 取 row（不存在 → DATASET_NOT_FOUND）
    2. 若 patch.label：dataset_importer.validate_segments(patch.label["segments"], row.duration_sec)
    3. UPDATE label / notes；updated_at 自動

async def delete_item(db, item_id: int) -> None:

    1. 取 row（找不到視為 idempotent 成功）
    2. file_store.delete(f"datasets/{item_id}/")  ← rmtree
    3. DELETE row
    """
```

## 4. Importer + Exporter

### 4.1 Importer：補完 `services/dataset_importer.py`

#### 共同流程

`import_label(label_path, audio_filename, audio_duration, format, project_hotwords)` 已定義（M3.5 stub），補完後流程：

```
PARSERS[format](label_path) → list[dict]（訓練 JSON segments，speaker 0-indexed）
↓
validate_segments(segments, audio_duration)  ← 已實作
↓
回傳 {
  "audio_duration": audio_duration,
  "audio_path": audio_filename,
  "segments": segments,
  "customized_context": project_hotwords,
}
```

#### `parse_json`（已實作，無變更）

假定已是 §9.1 schema，pass-through。

#### `parse_xlsx`（新增）

- 用 openpyxl `load_workbook(path, read_only=True)`，取第一個 sheet。
- header（第 1 列）必須含欄位 `start_time / end_time / speaker / text`（順序不限），缺欄 → `AppError(IMPORT_PARSE_FAILED, "Missing column: ...")`。
- 第 2 列起每列轉 `{start, end, speaker, text}`：
  - `start = parse_time(cell)` / `end = parse_time(cell)`（既有 `parse_time` 支援 float / hh:mm:ss / mm:ss）
  - `speaker = parse_speaker(cell)`（既有，1→0 indexed 轉換）
  - `text = str(cell).strip()`
  - 整列空白 → 跳過

#### `parse_txt`（新增）

行 regex：`^\[(\d+:\d{1,2}:\d{1,2}(?:\.\d+)?)\]\s+Speaker\s*(\d+)\s*[:：]\s*(.+)$`

- end_time 推算規則：下一行 start；最後一行用 `audio_duration`
- 不匹配的行：略過 + warning（log），不中斷
- 全部不匹配 / 空檔 → `AppError(IMPORT_PARSE_FAILED, "No valid lines parsed")`

#### `parse_srt`（新增）

用 pysrt：

```python
import pysrt
subs = pysrt.open(str(label_path), encoding="utf-8")
segments = []
for sub in subs:
    start = sub.start.ordinal / 1000.0  # ms → sec
    end   = sub.end.ordinal / 1000.0
    text  = sub.text.strip()
    m = re.match(r"^Speaker\s*(\d+)\s*[:：]\s*(.*)$", text, re.DOTALL)
    if m:
        speaker = max(0, int(m.group(1)) - 1)  # 1→0 indexed
        text    = m.group(2).strip()
    else:
        speaker = 0
    segments.append({"start": start, "end": end, "speaker": speaker, "text": text})
return segments
```

#### `_export_srt`

```

00:00:00,000 --> 00:00:03,450
Speaker 1: 各位早安

2
00:00:03,450 --> 00:00:07,200

```

#### `_export_xlsx`

- header `start_time / end_time / speaker / text`
- 時間欄寫 `seconds_to_hms(s, with_ms=True)`（hh:mm:ss.cc）
- speaker 寫 1-indexed int
- content_type `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- filename `dataset-{item.id}.xlsx`

route 層用：

```python
content, ct, name = export_item(item, format)
return Response(
    content=content, media_type=ct,
    headers={"Content-Disposition": f'attachment; filename="{name}"'},
)
```

#### 5.5.1 新檔 `frontend/src/lib/editorSource.ts`

```typescript
export interface EditorSource {
  load(): Promise<{
    segments: Segment[];        // UI 內部 1-indexed speaker
    audioUrl: string;
    durationSec: number;
    title: string;              // 標頭顯示用，例如 "Job ab123" 或 "Dataset #12"
  }>;
  save(snapshot: Segment[]): Promise<void>;
  // 不回傳 segments — 沿用 CLAUDE.md §4.12「以 frontend snapshot 當 originalSnapshot」
}

export function jobEditorSource(jobId: string, project: Project): EditorSource {
  return {
    async load() {
      const job = await jobsApi.get(jobId);
      return {
        segments: job.segments ?? [],
        audioUrl: jobsApi.audioUrl(jobId),
        durationSec: job.duration_sec ?? 0,
        title: `Job ${jobId}`,
      };
    },
    async save(snapshot) {
      await jobsApi.patchSegments(jobId, snapshot);
    },
  };
}

export function datasetEditorSource(itemId: number, project: Project): EditorSource {
  return {
    async load() {
      const item = await datasetsApi.get(itemId);
      // label.segments speaker 0→1 indexed for UI
      const segments = item.label.segments.map((s, idx) => ({
        idx,
        speaker_id: s.speaker + 1,
        text: s.text,
        start_time: s.start,
        end_time: s.end,
      }));
      return {
        segments,
        audioUrl: datasetsApi.audioUrl(itemId),
        durationSec: item.duration_sec,
        title: `Dataset #${itemId}`,
      };
    },
    async save(snapshot) {
      // segments speaker 1→0 indexed for DB
      const labelSegments = snapshot.map(s => ({
        speaker: s.speaker_id - 1,
        text: s.text,
        start: s.start_time,
        end: s.end_time,
      }));
      // 取出原 label 其餘欄位，僅替換 segments
      const item = await datasetsApi.get(itemId);
      const newLabel = { ...item.label, segments: labelSegments };
      await datasetsApi.patch(itemId, { label: newLabel });
    },
  };
}
```

#### 5.5.2 `editorStore.ts` 改動

- `init(source: EditorSource)` 取代直接吃 segments；store 內部呼叫 `source.load()` 並 set state。
- `save()` 內部呼叫 `source.save(segments)`。
- 其他（dirty 判定、clamp、resizeSegment、patchSegment、markSaved）**不變**。

#### 5.5.3 `TranscriptEditor.tsx` 改動

- props 從 `{job, audioUrl, projectId}` 改為 `{source: EditorSource, project: Project}`。
- 標頭 title 用 `source.load()` 回的 `title`。
- save 流程不變（仍 `useEditorStore.getState()` 拿最新 segments，遵循 CLAUDE.md §4.12）。
- shortcut / dirty toolbar / waveform region drag clamp 行為完全不變。

## 6. 錯誤處理

### 6.1 後端錯誤情境矩陣

| 觸發點 | 錯誤碼 | HTTP | 訊息範例 |
|---|---|---|---|
| `format` ∉ {json, xlsx, srt, txt} | UNSUPPORTED_FORMAT | 400 | `Format must be one of [json, xlsx, srt, txt]` |
| 標註檔解析失敗（XLSX 缺欄、SRT 結構壞） | IMPORT_PARSE_FAILED | 400 | `Missing column: text` |
| 時間欄無法解析 | IMPORT_INVALID_TIME | 400 | `Cannot parse time: 'foo'` |
| segments 為空 / overlap / 超出 audio_duration | IMPORT_PARSE_FAILED | 400 | `Segment 5: end 120.0 exceeds audio duration 100.0` |
| audio 無法取 duration（非音檔 / 損毀） | AUDIO_DURATION_FAILED | 400 | `Cannot extract duration from upload.bin` |
| `from_job` 對應 Job 不存在 | NOT_FOUND | 404 | `Job ab12 not found` |
| `from_job` Job status ≠ SUCCEEDED | INVALID_JOB_STATE | 400 | `Job ab12 is in failed, expected SUCCEEDED` |
| GET / PUT / DELETE 對不存在 item | DATASET_NOT_FOUND | 404 | `Dataset item 99 not found` |
| PUT 的 `label.segments` 違反 validate | IMPORT_PARSE_FAILED | 400 | 同 import |
| audio 上傳超過 size limit | UPLOAD_TOO_LARGE | 413 | sanity 訊息 |

### 6.2 回滾策略（service 層）

`create_from_import` / `create_from_job` 中途失敗：

- DB row 已 INSERT 但 audio 複製失敗 → 手動 `await delete_item(db, item_id)` 清理
- 用 try/except + rollback handler，避免半建立狀態

`delete_item` 找不到 row 視為 idempotent（不報 404）— 與 RESTful 慣例一致。

### 6.3 前端錯誤呈現

| 場景 | 處理 |
|---|---|
| 4xx 業務錯誤 | `client.ts` 既有：抽 `detail.detail` 進 toast |
| 5xx | toast「服務異常」 |
| 匯入 modal 中途失敗 | modal 不關（保留 user 已選檔），上方 inline error bar 顯示細節 |
| Editor save 失敗 | 既有 toast；dirty 不 reset，使用者可重試 |
| 從歷史轉錄 modal Job list 載入失敗 | inline「載入失敗，重試」按鈕 |

## 9. 不變式（Invariants）

實作期間必須持守：

1. `DatasetItem.label.segments[].speaker` **永遠是 0-indexed int**。
2. Speaker 索引轉換**只發生於**：dataset_importer.parse_speaker / dataset_service.create_from_job / dataset_exporter（SRT、XLSX）/ datasetEditorSource（load、save）。
3. `routes/admin/datasets.py` **不寫業務邏輯** — 只負責 multipart 解析、依賴注入、呼叫 service、Pydantic 序列化、`http_error()` 包錯。
4. `dataset_importer` / `dataset_exporter` **純 stateless 函式** — 不沾 DB、不沾 file_store。
5. `dataset_service` **不沾 parsing / 不沾格式** — 純 ORM + file_store。
6. Editor 既有行為（CLAUDE.md §4.12）**不可破壞**：`save()` 用 `getState()`、`markSaved` 用 frontend snapshot、time inputs 與 region drag 共用 `resizeSegment`。
7. CSV / VTT 不在範圍內，但 `format` enum 與 `PARSERS` dict 須能輕易加回（不寫死 if/else）。
